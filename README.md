# Hello World App

## Описание проекта

Простое веб-приложение, отображающее текст «Hello World» и две кнопки для изменения цвета текста на зелёный и оранжевый.

## Используемые технологии

### Frontend

В проекте используются следующие технологии:

* HTML — создание структуры веб-страницы;
* CSS — оформление интерфейса;
* JavaScript — реализация логики изменения цвета текста.

### Framework

Фреймворки в данном проекте не используются, так как для решения поставленной задачи достаточно стандартных возможностей HTML, CSS и JavaScript.

### Backend

Backend отсутствует.

Приложение является статическим и работает полностью в браузере пользователя. Для выполнения задания не требуется серверная часть, база данных или хранение данных.

## Причины выбора технологий

* HTML обеспечивает структуру страницы;
* CSS отвечает за внешний вид элементов;
* JavaScript реализует обработку нажатий на кнопки;
* отсутствие Backend упрощает разработку и развертывание приложения.


## Возможные уязвимости приложения при публикации в публичном доступе

Несмотря на простоту приложения, при его публикации в интернете следует учитывать возможные риски безопасности.

### 1. Изменение клиентского кода

Все файлы приложения (HTML, CSS и JavaScript) загружаются в браузер пользователя. Пользователь может просматривать и изменять код через инструменты разработчика (DevTools).

В данном проекте риск является несущественным, так как приложение не содержит конфиденциальных данных и не выполняет критически важные операции.

### 2. Отсутствие HTTPS

Если приложение размещено по протоколу HTTP, злоумышленник может перехватить или изменить передаваемые данные.

Для защиты рекомендуется использовать HTTPS и действующий SSL-сертификат.

### 3. Возможность XSS-атак в будущем

В текущей версии приложения пользовательский ввод отсутствует. Однако при дальнейшем развитии проекта может появиться риск XSS (Cross-Site Scripting), если данные пользователя будут выводиться на страницу без проверки.

Для предотвращения подобных атак необходимо выполнять проверку и экранирование пользовательских данных.

### 4. Публикация конфиденциальной информации

При размещении проекта в публичном репозитории существует риск случайной публикации паролей, API-ключей и других секретных данных.

Для предотвращения данной проблемы рекомендуется не хранить конфиденциальную информацию в исходном коде и использовать файл `.gitignore`.

### 5. Использование устаревших библиотек

При подключении сторонних библиотек в будущем возможно появление известных уязвимостей в используемых версиях зависимостей.

Для снижения риска необходимо регулярно обновлять используемые библиотеки и следить за сообщениями о безопасности.
